# 📋 kdy-addon 제안서 평가 보고서

## 1. 프로젝트 현황 분석 (Context)
현재 프로젝트는 **"AI 기반 코인 차트 분석 및 시그널 제공"**이라는 핵심 기능을 강력하게 구축한 상태입니다.
- **강점**: `analysis.ts`, `fractal_engine.ts` 등 독창적이고 고도화된 분석 로직 보유.
- **현황**: 누구나 접근 가능한 **Public 서비스** 형태. 사용자 개개인의 데이터(포트폴리오, 알림 설정, 내역 등)를 저장하고 관리하는 **Private 영역**이 부재함.
- **기술적 기반**: Next.js 16 + Supabase 라이브러리는 설치되어 있어, 확장을 위한 기초는 마련됨.

---

## 2. 제안서별 평가

### 📄 Proposal 01: 서비스 구조화 및 인증 (Auth & Personalization)
> **핵심 내용**: 구글/깃허브 로그인 도입, 개인화 페이지(포트폴리오, 설정) 분리, DB(Profiles, Trades) 설계.

- **평가**: ⭐⭐⭐⭐⭐ **(최우선 순위)**
- **이유**:
  1. **필수 인프라**: 사용자 맞춤형 기능(알림, 포트폴리오, 승률 기록)을 제공하려면 `User ID` 식별이 선행되어야 합니다.
  2. **가치 상승**: 단순 "조회용 도구"에서 "나만의 트레이딩 비서"로 서비스 가치가 격상됩니다.
  3. **준비 상태**: 이미 `supabase-js`가 설치되어 있고, 제안서에 구체적인 구현 코드(미들웨어, RLS 정책)가 포함되어 있어 즉시 적용 가능합니다.
- **리스크**: 낮음 (표준적인 OAuth 구현).

### 📄 Proposal 02 & 03: 게이미피케이션 및 소셜 (Gamification & Social)
> **핵심 내용**: 트레이딩 배틀, 예측 챌린지, 승률 리더보드, 성과 카드 공유, AI 챗봇.

- **평가**: ⭐⭐⭐ **(2단계 확장팩)**
- **이유**:
  1. **높은 잠재력**: 사용자의 **리텐션(재방문율)**을 획기적으로 높일 수 있는 기능들입니다. 특히 '예측 챌린지'나 '성과 카드'는 사용자가 자발적으로 서비스를 홍보하게 만듭니다.
  2. **의존성**: 대부분의 기능(리더보드, 예측 기록, 뱃지)이 **"로그인한 사용자"**를 전제로 합니다. Proposal 01 없이 진행하면 데이터가 휘발되어 의미가 퇴색됩니다.
  3. **구현 복잡도**: 실시간성(랭킹), 소셜 기능 등은 개발 공수가 큽니다.
- **추천**: Proposal 01 완료 후, 가장 효과적인 기능 1~2개(예: 예측 챌린지)부터 점진적으로 도입하는 것이 좋습니다.

## 3. 종합 추천 (Recommendation)

**"Proposal 01(회원가입/로그인)을 먼저 도입하여 서비스의 뼈대를 완성한 후, 그 위에 Proposal 03(게이미피케이션)의 예측 기능을 얹는 단계적 접근을 권장합니다."**

### 🚀 추천 로드맵
1.  **Phase 1 (기반 구축)**: `pp01.txt` 적용
    *   Supabase Auth 연동 (Google 로그인)
    *   개인화 DB 구축 (Profiles, Trades)
    *   나만의 포트폴리오/관심종목 페이지 오픈
2.  **Phase 2 (재미 요소 추가)**: `pp03.txt` 중 'Prediction Challenge' 도입
    *   로그인한 유저가 코인 등락 예측
    *   적중 시 포인트/뱃지 부여 (Phase 1의 DB 활용)

**결론**: 현재 프로젝트는 "똑똑한 도구" 단계입니다. 이를 "사용자 중심의 플랫폼"으로 진화시키기 위해 **Proposal 01의 적용이 시급**합니다.
